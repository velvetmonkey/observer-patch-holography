# OPH Negative Results Registry

This registry is the public home for failed branches, rejected parameterizations, null hardware runs, nonconfluent examples, MAR competitors, and alternative fixed points.

Each entry should record:

- stable identifier
- affected claim ID from `claims/claim_registry.yaml`
- source commit
- attempted branch or proof route
- exact failure mode
- raw artifact or verifier path
- consequence for the claim tier
- parts of the stack that survive

The absence of entries in this seed file is not evidence for any theorem. It means no negative-result packet has been registered in this directory.
